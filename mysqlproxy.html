Test de MySQL Proxy

Utilise Lua.
Compatibilité OS : Linux, Mac OS X, Free BSD, Windows XP/Vista, Windows Server 2003/2008.
Installation sous Debian : sudo aptitude install mysql-proxy
Possibilité d'écrire la config dans un fichier.
On le spécifiera dans la ligne de commande avec --defaults-file=file_name.
Il doit avoir les permissions 660 au minimum.
http://dev.mysql.com/doc/refman/5.6/en/mysql-proxy-configuration.html.
Le fichier sera de la forme :

[mysql-proxy]
admin-address = host:port
admin-password = password
daemon = true

Mais possibilité également de faire tout par la ligne de commande :
mysql-proxy --admin-address=host:port --admin-password=password --daemon

Fonctions à redéfinir dans le script Lua :

  • connect_server() appelée à chaque connexion d'un utilisateur. Utile pour le loadbalancing. http://dev.mysql.com/doc/refman/5.6/en/mysql-proxy-scripting-connect-server.html.
  • read_handshake() appelée au premier handshake retourné par le serveur. Utile pour ajouter des vérifications. http://dev.mysql.com/doc/refman/5.6/en/mysql-proxy-scripting-read-handshake.html.
  • read_auth() appelée quand le paquet de connexion (db, user, password) est envoyé au serveur. http://dev.mysql.com/doc/refman/5.6/en/mysql-proxy-scripting-read-auth.html.
  • read_auth_result() appelée quand le serveur retourne l'autorisation. http://dev.mysql.com/doc/refman/5.6/en/mysql-proxy-scripting-read-auth-result.html.
  • read_query() appelée à chaque requête envoyée par le client. Utile pour éditer ou filtrer les requêtes. http://dev.mysql.com/doc/refman/5.6/en/mysql-proxy-scripting-read-query.html.
  • read_query_result() appelée à chaque résultat de requête (préalablement injectée dans la queue avec read_query()). http://dev.mysql.com/doc/refman/5.6/en/mysql-proxy-scripting-read-query-result.html.

Bien veiller à ce qu'il y ait le même nombre de requêtes que de resultsets.
Le proxy contient les attributs connection (contient les attributs client et server), servers, queries, response.
Exemple pour ajouter une requête à la queue :
proxy.queries:append( 1, packet, { resultset_is_needed = true } )
Le premier paramètre est l'id souhaité, le deuxième le paquet contenant la requête, et le dernier permettra de déclencher la fonction read_query_result().
Si on le met à false, le résultat sera directement envoyé et donc ce sera plus rapide.
Possibilité de redéfinir la response avec son type (packet ou error) et son contenu (resultset ou string).
Exemple : proxy.response.resultset = { fields = { ... }, rows = { ... } }
